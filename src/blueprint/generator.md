# Blueprint Generator

Instructions for Claude Code to generate a setup blueprint from interview answers.

## Input

- `project-context.json` at the target project directory (created by interviewer)
- Preset files at `~/.launchpad/presets/{project_type}/`

## Flow

### Step 1: Load context

1. Read `{project_path}/project-context.json`
2. Extract `project_type` from the context

### Step 2: Load preset

Read these files from the matching preset directory:

| File | Purpose |
|------|---------|
| `presets/{project_type}/preset.json` | Default config, directory list, module order |
| `presets/{project_type}/skills.json` | Skills to install |
| `presets/{project_type}/agents.json` | Agents to generate |
| `presets/{project_type}/standards/` | Code standard templates (if any) |
| `presets/{project_type}/rules/` | Rule templates (if any) |

If `project_type` is `custom`, use minimal defaults: only core modules, no preset-specific skills or standards.

### Step 3: Merge config

Build the final config by merging (user answers override preset defaults):

```
final_config = {
  ...preset.json defaults,
  ...project-context.json answers,
  skills: [...preset skills, ...additional_skills from user],
  modules: preset.modules (filtered by user's choices)
}
```

Filtering rules:
- If `repo_status == existing`: mark `repo-setup` as skip (repo already exists)
- If `need_pm == false`: mark `pm-setup` as skip
- If `need_memory == false`: mark `memory-setup` as skip
- If `team_mode == solo`: skip agent-teams-specific config in `agents-generator`

### Step 4: Generate blueprint.md

Create the blueprint file with this structure:

```markdown
# Blueprint: {project_name}

Generated by Launchpad on {timestamp}

## Project overview

| Field      | Value                    |
|------------|--------------------------|
| Name       | {project_name}           |
| Type       | {project_type}           |
| Stack      | {tech_stack}             |
| Repo       | {repo_status} / {repo_visibility} |
| Team       | {team_mode}              |
| PM         | {need_pm}                |
| Memory     | {need_memory}            |

## Modules

| #  | Module              | Status  | Description                              |
|----|---------------------|---------|------------------------------------------|
| 1  | repo-setup          | include | Initialize git repo and GitHub remote    |
| 2  | claude-settings     | include | Generate .claude/settings.json           |
| 3  | rules-generator     | include | Generate CLAUDE.md with project rules    |
| 4  | code-standards      | include | Install code standards from preset       |
| 5  | skills-installer    | include | Install skills from registry             |
| 6  | agents-generator    | include | Generate AGENTS.md and agent configs     |
| 7  | memory-setup        | include | Initialize Claude Code memory system     |
| 8  | pm-setup            | include | Create plans/ and docs/ structure        |
| 9  | templates-scaffold  | include | Scaffold project files and directories   |
| 10 | workspace-setup     | include | Configure IDE and dev environment        |

(Status is "include" or "skip" based on Step 3 filtering)

## Module details

### 1. repo-setup
- **Action:** git init, create .gitignore ({gitignore_template}), gh repo create
- **Config:** visibility={repo_visibility}, name={project_name}
- **Creates:** .git/, .gitignore

### 2. claude-settings
- **Action:** Generate Claude Code settings with allowed/denied tools
- **Source:** preset.json
- **Creates:** .claude/settings.json

### 3. rules-generator
- **Action:** Generate CLAUDE.md with project-specific rules
- **Source:** presets/{project_type}/rules/, project-context.json
- **Creates:** CLAUDE.md

### 4. code-standards
- **Action:** Copy and adapt code standards for the project
- **Source:** presets/{project_type}/standards/
- **Creates:** docs/code-standards.md

### 5. skills-installer
- **Action:** Install skills from registry sources
- **Source:** presets/{project_type}/skills.json, registry/skills-sources.json
- **Skills:** {list each skill name}
- **Creates:** .claude/skills/{skill-name}/SKILL.md for each skill

### 6. agents-generator
- **Action:** Generate agent definitions
- **Source:** presets/{project_type}/agents.json
- **Agents:** {list each agent name + role}
- **Creates:** AGENTS.md

### 7. memory-setup
- **Action:** Initialize memory directory and MEMORY.md index
- **Creates:** ~/.claude/projects/.../memory/MEMORY.md

### 8. pm-setup
- **Action:** Create project management directories and templates
- **Creates:** plans/, plans/reports/, docs/

### 9. templates-scaffold
- **Action:** Create project source directories and starter files
- **Directories:** {list from preset.json directories}
- **Creates:** Directory structure + placeholder files

### 10. workspace-setup
- **Action:** Configure IDE settings and dev environment
- **Creates:** .vscode/settings.json, .env.example (if applicable)

## Summary

- Modules to run: {count of included modules}/10
- Skills to install: {count}
- Agents to generate: {count}
```

### Step 5: Save blueprint

Save the generated blueprint to:

```
{project_path}/.claude/launchpad/blueprint.md
```

Create the `.claude/launchpad/` directory if it does not exist.

### Step 6: Present for review

Display the blueprint to the user and ask:

"Approve this blueprint? (yes / edit / cancel)"

- **yes** - proceed to Phase 4 (module execution) per SKILL.md
- **edit** - ask what to change. Supported edits:
  - Toggle a module between include/skip
  - Add or remove skills from the skills list
  - Change project config values
  - After edits, regenerate blueprint.md and ask again
- **cancel** - abort, keep blueprint.md for reference but do not proceed

## Rules

- Always show the full module table so user sees what will and will not run
- Do not execute any module during blueprint generation - this is planning only
- If a preset file is missing, warn the user and use sensible defaults
- Translate all user-facing text to the user's detected language
- Keep blueprint.md in English regardless of user language (it is a config artifact)
